# CUDA 并行归约优化

本项目实现了 5 个版本的 CUDA 并行归约算法，逐步优化性能。

## 性能测试结果

测试配置：
- 数据规模：N = 32M (32 * 1024 * 1024)
- Block 大小：256 threads
- 理论带宽：1008 GB/s
- 测试次数：每个版本运行 20 次取平均

| 版本 | 耗时 (ms) | 带宽 (GB/s) | 带宽效率 | 正确性 |
|------|-----------|-------------|----------|--------|
| v0   | 0.386     | 349.21      | 34.64%   | PASS   |
| v1   | 0.340     | 396.30      | 39.32%   | PASS   |
| v2   | 0.348     | 387.33      | 38.43%   | PASS   |
| v3   | 0.241     | 558.62      | 55.42%   | PASS   |
| v4   | 0.151     | 891.96      | 88.49%   | PASS   |

## 各版本优化说明

### v0: 基础全局内存版本
- 直接使用全局内存进行归约
- 存在大量全局内存访问，性能较差
- 带宽效率：34.64%

### v1: 共享内存优化
- 使用共享内存减少全局内存访问
- 每个 block 先将数据加载到共享内存
- 带宽效率提升至 39.32%

### v2: 解决 Warp Divergence
- 优化线程分支，减少 warp 内的分支发散
- 改进归约时的线程索引方式
- 带宽效率：38.43%（略有下降，可能是其他因素）

### v3: 解决 Bank Conflict
- 优化共享内存访问模式，避免 bank conflict
- 使用连续的线程 ID 访问共享内存
- 带宽效率大幅提升至 55.42%

### v4: 加载时归约（最优版本）
**核心优化点：**

1. **每个 block 处理 2 倍数据量**
   - v0-v3：每个 block 处理 256 个元素
   - v4：每个 block 处理 512 个元素（2 * blockDim.x）

2. **加载时完成第一轮归约**
   - 每个线程加载 2 个元素并立即求和：`sdata[tid] = input[tid] + input[tid + blockDim.x]`
   - 数据加载到共享内存时已完成 512→256 的归约

3. **性能提升原因**
   - 所有线程都参与计算，无线程闲置（v3 第一轮只有一半线程工作）
   - 充分利用内存带宽，每个线程加载 2 个数据
   - 减少 block 数量（减半），降低 block 启动和同步开销
   - 带宽效率达到 **88.49%**，接近理论峰值

4. **性能对比**
   - 相比 v3 提升：558.62 → 891.96 GB/s（**59.7% 提升**）
   - 相比 v0 提升：349.21 → 891.96 GB/s（**155.4% 提升**）

## 编译和运行

### 编译单个版本
```bash
make reduce_v0  # 编译 v0
make reduce_v1  # 编译 v1
# ... 以此类推
```

### 运行 benchmark 对比所有版本
```bash
make benchmark
./benchmark
```

### 清理
```bash
make clean
```
